# android_resource()

::: overview
An `android_resource()` rule is used to bundle Android resources that
are traditionally stored in `res` and `assets` directories.

The output of an `android_resource()` is an `R.txt` file generated via
`aapt --output-text-symbols`.

## Arguments

-   ::: {#name}
    ### `name`{.argName} [(required)]{.argDefault} [\#](#name){.inline-link}

    The *short name* for this [build
    target](/concept/build_target.html).
    :::

-   ::: {#res}
    ### `res`{.argName} [(defaults to `None`)]{.argDefault} [\#](#res){.inline-link}

    A dictionary mapping relative resource paths to either the resource
    files or the build targets that generate them. The
    [`subdir_glob()`](/function/subdir_glob.html) function can be used
    to generate dictionaries based on a directory structure of files
    checked into the repository. Alternatively, this can be a path to a
    directory containing Android resources, although this option is
    deprecated and might be removed in the future.
    :::

-   ::: {#package}
    ### `package`{.argName} [(defaults to `None`)]{.argDefault} [\#](#package){.inline-link}

    Java package for the `R.java` file that will be generated for these
    resources.
    :::

-   ::: {#assets}
    ### `assets`{.argName} [(defaults to `None`)]{.argDefault} [\#](#assets){.inline-link}

    A dictionary mapping relative asset paths to either the asset files
    or the build targets that generate them. The
    [`subdir_glob()`](/function/subdir_glob.html) function can be used
    to generate dictionaries based on a directory structure of files
    checked into the repository. Alternatively, this can be a path to a
    directory containing Android assets, although this option is
    deprecated and might be removed in the future.
    :::

-   ::: {#project_res}
    ### `project_res`{.argName} [(defaults to `None`)]{.argDefault} [\#](#project_res){.inline-link}

    A directory containing resources to be used for project generation.
    If not provided, defaults to whatever the build uses.
    :::

-   ::: {#project_assets}
    ### `project_assets`{.argName} [(defaults to `None`)]{.argDefault} [\#](#project_assets){.inline-link}

    A directory containing assets to be used for project generation. If
    not provided, defaults to whatever the build uses.
    :::

-   ::: {#manifest}
    ### `manifest`{.argName} [(defaults to `None`)]{.argDefault} [\#](#manifest){.inline-link}

    An optional [Android
    Manifest](http://developer.android.com/guide/topics/manifest/manifest-intro.html)
    for the resource to declare any permissions or intents it may need
    or want to handle. May either be a file or a
    [`android_manifest`](/rule/android_manifest.html) target.
    :::

-   ::: {#deps}
    ### `deps`{.argName} [(defaults to `None`)]{.argDefault} [\#](#deps){.inline-link}

    Other `android_resource` rules to include via `-S` when running
    `aapt`.
    :::

-   ::: {#visibility}
    ### `visibility`{.argName} [(defaults to `[]`)]{.argDefault} [\#](#visibility){.inline-link}

    List of [build target patterns](/concept/build_target_pattern.html)
    that identify the build rules that can include this rule as a
    dependency, for example, by listing it in their `deps` or
    `exported_deps` attributes. For more information, see
    [visibility](/concept/visibility.html).
    :::

-   ::: {#licenses}
    ### `licenses`{.argName} [(defaults to `[]`)]{.argDefault} [\#](#licenses){.inline-link}

    Set of license files for this library. To get the list of license
    files for a given [build rule](/concept/build_rule.html) and all of
    its dependencies, you can use [`buck query`](/command/query.html).
    :::

-   ::: {#labels}
    ### `labels`{.argName} [(defaults to `[]`)]{.argDefault} [\#](#labels){.inline-link}

    Set of arbitrary strings which allow you to annotate a [build
    rule](/concept/build_rule.html) with tags that can be searched for
    over an entire dependency tree using
    [`buck query attrfilter()`](/command/query.html#attrfilter).
    :::

## Examples

Most of the time, an `android_resource` rule defines only `name`, `res`,
and `package`. By convention, such simple rules are often named `res`:

``` {.prettyprint .lang-py}
android_resource(
  name = 'res',
  res = subdir_glob([('res', '**')]),
  package = 'com.example',
)
```
